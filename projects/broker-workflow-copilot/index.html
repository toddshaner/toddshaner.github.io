Project 02

Broker Workflow Copilot

A prototype for helping insurance producers and account teams generate structured briefs, renewal preparation notes, and meeting prep materials from mock client information.

Why this matters

A large share of brokerage work depends on collecting scattered information, synthesizing it quickly, and turning it into usable preparation for client conversations and internal execution. This project explores where AI can reduce prep time while preserving human judgment.

Core use case

Given a mock client profile and supporting materials, the copilot should help produce an account brief, renewal preparation notes, meeting prep prompts, and follow-up task suggestions.

MVP scope

  • Take a narrow set of mock inputs
  • Generate a repeatable account brief template
  • Identify missing information
  • Suggest questions for renewal or client discussion
  • Preserve a visible human-review step before use

Example inputs

  • Mock company profile
  • Mock lines of coverage
  • Prior renewal notes
  • Mock claims summary
  • Internal meeting notes

Example outputs

  • Executive account summary
  • Exposure overview
  • Renewal timeline checklist
  • Open issues and missing data list
  • Discussion questions for producer or account manager

Human oversight and governance

This tool drafts; it does not replace producer judgment. Client-facing recommendations remain human-owned. The workflow should clearly separate generated draft content from approved final material and use mock or synthetic data only.

Next steps

Add a UI for input selection and brief generation, create reusable prompt templates, define evaluation criteria for usefulness and completeness, and support multiple output formats.
