Portfolio

Projects built around real insurance workflows.

The portfolio is intentionally narrow. Each project is designed to connect frontier model capability to a concrete insurance task, with clear boundaries for human review and governance.

How I approach builds
1. Choose a narrow workflowStart with a task that matters operationally.
2. Design for reviewSeparate draft assistance from final judgment.
3. Evaluate usefulnessMeasure whether the workflow is actually better, not just novel.

Portfolio rule

No confidential or real client data. All demos use mock or synthetic material.

For insurance use cases, credibility depends on both relevance and restraint. That means good workflow design, clear caveats, and clean repository hygiene.
